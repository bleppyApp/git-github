GIT Commands

  • git status

    gives status on a git repository and its contents

  • git init

    creates a new repository for the current folder. Before initialize a repo, run git status first to make sure that the current location is not being watched by git.

  • git add

    stages changes before you can commit them. To stage multiple files at once, separate files with spaces, like: git add file1 file2

  • git add .

    stages all changes at once

  • git rm --cached fileName

    unstages a staged file

  • git commit -m "my message about the commitment"

    moves the staged changes from the staging area to the repository.

  • git commit -a -m "my message about the commitment"

    shortcut for staging and committing all files at once.

  • git commit --amend

    allows you to add a forgotten file(s) to the last commit or edit the message from the last commit. If the idea is to add a file, you'll have to stage it first with get add.

  • git branch

    gives the list of existing branches. And the one with an * is the active branch

  • git branch name_of_branch

    creates a new branch but it does not switch to it.

  • git switch name_of_branch

    goes to the named branch

  • git switch -c name_of_branch

    creates a new branch and navigates to it at once

  • git branch -m newName

    modifies/renames the current branch

  • git branch -d name_of_branch

    deletes branch. However, to make this work, this branch has be merged first. If you don't want to merge it, you can force the deletion by using uppercase d, -D, instead of -d
